# Cat Note Illustrations

## 核心定位

为中文文章和飞书笔记设计、规划并生成 16:9 横版正文配图。目标不是商业插画、PPT 信息图或复杂架构图，而是把正文里的关键判断、流程、结构、状态或隐喻，转译成一张纯白背景、清爽可读、带猫猫动作隐喻的卡通贴纸式插图。

默认视觉 IP 是 Specimen 0：一只米白身体、深色头顶/后背/尾巴斑块、明黄色大眼、黑色细竖瞳的双色短毛猫。它的气质是“清空大脑但看透一切”的呆滞幽默感，画风是手账便签贴纸感的中等粗黑手绘线、低饱和暖色平涂和极淡色铅笔/纸感颗粒。猫猫必须参与画面的核心动作，不能只是坐在旁边卖萌。

## 来源说明

这个 Skill 基于 `helloianneo/ian-xiaohei-illustrations` 改造而来，保留“先理解正文认知锚点，再生成正文配图”的工作流思想，但默认视觉 IP、风格 DNA、提示词和 QA 标准已改成猫猫贴纸画风。发布或上传 GitHub 时，请保留 `ATTRIBUTION.md`。

## 工作流

### 1. 消化正文

先读用户给的正文、飞书笔记、Markdown、链接文本或截图内容。提炼：

- 核心观点是什么
- 哪些段落承担认知转折
- 哪些内容适合用图解释
- 哪些地方只适合文字，不需要图

不要平均配图。优先选择“认知锚点”，例如：核心判断、两个断点、输入输出闭环、分流、前后对比、一鱼多吃、承接路径、常见坑、角色状态变化。

### 2. 先出配图策略

如果用户只是说“分析怎么配图 / 思考哪些地方需要配图”，先给 shot list。每张图写清楚：

- 放在哪个段落后
- 图的主题
- 核心意思
- 结构类型
- 猫猫在图里做什么
- 建议元素
- 建议中文标注词

默认 4-8 张。文章很短时 1-3 张；长文也不要轻易超过 9 张。够用就好，避免把正文做成画册。

### 3. 单张生成

如果用户明确要求“生成 / 输出 / 做图 / 帮我生成”，不要停下来等确认；用内置 `image_gen` 每张单独生成。不要把多张图拼在一张里。

每张图只讲一个核心结构。提示词必须包含：

- 16:9 横版中文正文配图
- 纯白背景，无纸纹、无阴影、无渐变
- 均匀黑色手绘轮廓线，闭合色块
- Specimen 0 双色猫作为核心动作主体，保留黄色大眼和细竖瞳
- 低饱和暖色猫毛色系，平涂并带极淡纸感/色铅笔颗粒
- 少量同色系短线毛发纹理
- 极少中文短标注，必要时才出现
- 禁止 PPT、商业矢量、复杂背景、写实光影、密集文字、复刻参考图

不要复刻参考图片的具体猫、姿势、斑纹或构图。参考只用于校准 Specimen 0 的角色 DNA：双色斑块、黄色竖瞳、手账贴纸线条、低饱和平涂、稀疏毛发短线和呆滞幽默感。每次都要从当前文章重新发明一个适合正文的猫猫动作隐喻。

### 4. 检查与迭代

生成后检查 `references/qa-checklist.md`。如果出现以下问题，优先重生成或局部编辑：

- 猫猫只是装饰，没有承担核心动作
- 画面太满或文字太多
- 太像流程图/PPT/课程页
- 猫咪变写实、3D、厚涂或商业矢量
- 线条太细碎、太抖、没闭合，或色块没有被黑线包住
- 背景不是干净白底
- 中文错字严重或标注不可读

### 5. 保存交付

如果用户在 workspace 内工作，把最终图复制到：

```text
assets/<article-slug>-illustrations/
```

按顺序命名：

```text
01-topic-name.png
02-topic-name.png
```

保留原始生成文件，不要覆盖已有资产，除非用户明确要求替换。

## 用户话术原则

对外示例不要要求用户复述风格细节。普通用户只需要提供主题、文章或一句话，例如：`Use $cat-note-illustrations 为“某个观点”生成一张正文配图。` 风格细节由本 Skill 内部规则负责，不要在常规使用示例里追加“粗黑轮廓、三角耳、长椭圆眼”等调试型要求。
## 输出口径

生成前的策略输出要短而准。生成后的交付要包含：

- 生成了几张
- 每张图的用途
- 保存路径
- 哪些图最稳，哪些图是可选

不要长篇解释风格理论；让图自己说话。
